fix(ui): Button's asChild rendered a plain <span> instead of merging props onto its child

Every "Button asChild" in the app (landing page CTAs, the shared
BackButton, the user-menu trigger) was wrapping its child element
(usually a react-router Link) in an extra <span>, instead of merging
the button's classes directly onto that child the way shadcn's
asChild pattern is supposed to work. Concretely this meant:

- The actual clickable <a> never received the button's layout classes
  (inline-flex items-center, gap, etc), so icon + label could render
  inconsistently depending on the surrounding flex context — visible
  as the landing page's "Explore Demo" button stacking its play icon
  above the label instead of inline next to it, unlike the primary
  CTA beside it.
- Keyboard focus rings never appeared on these buttons: focus-visible
  styling was applied to the non-interactive <span> wrapper, while
  actual keyboard focus lands on the inner <a>, which had none of
  that styling.

@radix-ui/react-slot wasn't even installed — the component silently
used a plain string tag instead. Added the real dependency and swapped
in Radix's Slot, which is the standard, correct way to implement
asChild. Verified every asChild call site in the app passes exactly
one child element (Slot's requirement).
